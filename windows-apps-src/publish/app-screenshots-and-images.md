# <a name="app-screenshots-images-and-trailers"></a>应用屏幕截图、图像和预告片

设计精良的图像是在应用商店中向潜在客户表示应用的主要途径之一。

你可以提供要在应用的应用商店一览中包含的[屏幕截图](#screenshots)、[徽标](#store-logos)和其他插图资源（如[预告片](#trailers)和[促销图像](##additional-art-assets)）。 这些项目中的一些是必要的，一些是可选的（不过一些可选图像应该包含在内，以获得最佳的应用商店显示）。 

在[应用提交过程](app-submissions.md)中，你将在[应用商店一览](create-app-store-listings.md)步骤中提供这些插图资源。 注意，应用商店中使用的图像和图像显示方式可能有所不同，具体取决于客户的操作系统以及其他因素。

应用商店还可能会使用你包含在应用程序包中的应用磁贴和其他图像。 提交应用前，请运行 [Windows 应用认证工具包](../debug-test-perf/windows-app-certification-kit.md)来确定你是否缺少任何必需的图像。 有关这些图像的指南和建议，请参阅[磁贴和图标资源](../controls-and-patterns/tiles-and-notifications-app-assets.md)。

## <a name="screenshots"></a>屏幕截图

屏幕截图是应用在应用商店一览中向客户显示的图像。

你可以选择为不同的设备系列提供屏幕截图，以便客户在该类型的设备上查看应用的应用商店一览时显示相应的屏幕截图。 

提交过程中只需要一张屏幕截图（对于任何设备系列），不过你也可以提供多张屏幕截图。最多可为桌面设备提供 9 张屏幕截图，而最多可为其他设备系列提供 8 张屏幕截图。 我们建议为应用支持的每一种设备系列至少提供 4 张屏幕截图，以便客户了解应用在其设备类型上的外观。

> [!NOTE]
> Microsoft Visual Studio 提供了一个[可帮助你捕获屏幕截图的工具](https://docs.microsoft.com/visualstudio/debugger/run-windows-store-apps-in-the-simulator#BKMK_Capture_a_screenshot_of_your_app_for_submission_to_the_Microsoft_Store)。

每张屏幕截图都必须是 .png 文件，采用横向或纵向布局，且文件大小不能大于 5 MB。

大小要求因设备系列而异：
- 移动设备：768 x 1280、720 x 1280 或 480 x 800 像素
- 桌面设备：1366 x 768 像素或更大
- 全息设备：1268 x 720 像素或更大
- Xbox：3480 x 2160 像素或更小

为了获得最佳的显示效果，创建屏幕截图时，请牢记以下指南：
- 将关键的视觉效果和文本保留在图像顶部的 3/4 区域中。 文本覆盖可以显示在底部的 1/4 区域中。 
- 请勿将其他徽标、图标或市场营销消息添加到你的屏幕截图中。
- 请勿使用非常浅或深的颜色，或高对比度的条纹，因为这样可能会干扰文本覆盖的可读性。

你还可以提供一段简短的描述文字，最多使用 200 个字符介绍每张屏幕截图。

> [!TIP]
> 屏幕截图会按顺序显示在你的应用商店一览中。 上传屏幕截图后，你可以拖放以对其重新进行排序。 

注意，如果你创建了[多种语言](supported-languages.md)的应用商店一览，你将为每种语言提供一个**应用商店一览**页面。 你将需要为每种语言单独上传图像（即使你使用相同的图像），并提供要用于每种语言的描述文字。


## <a name="store-logos"></a>应用商店徽标

应用商店徽标是可选的图像，你可以在应用商店中上传更加个性化的图像。 我们建议你提供这些图像，以在应用支持的所有设备和操作系统版本上获得最佳的应用商店一览显示效果。

你可以提供 .png 格式的图像文件，这些图像需要遵循以下指南：

- **9:16（720 x 1080 or 1440 x 2160 像素）**：这将在应用商店一览中作为主图像在 Windows 10 和 Xbox 设备上向客户显示，所以我们强烈建议提供该图像；如果没有该图像，一览效果可能不佳。 图像应包括你的应用名称，图像中的任何文本应满足可访问的可读性要求（4.51 对比率）。  
- **1:1（1080 x 1080 or 2160 x 2160 像素）**：这种图像可能会显示在某些布局中。 如果提供，请务必包括应用的名称。
- **1:1（300 x 300 像素）**：这种图像有时也称为**应用磁贴图标**，在 Windows Phone 8.1 及更早版本上向客户显示你的应用的应用商店一览时使用。 如果不提供此图像，Windows Phone 8.1 或更早版本上的客户将在你的应用一览中看到空白图标。 （如果你的应用仅有面向 Windows Phone 8.1 或更早版本的软件包，这也适用于 Windows 10 客户。）如果你的提交*仅*包括 UWP 软件包，则无需提供此图像。 （请注意，如果你的提交包含 Windows Phone 8.x 包和 UWP 程序包，并提供此图像，它可能会用在 Windows 10 的某些应用商店布局中。 为了防止这种情况，你可以为 Windows Phone 操作系统版本创建[特定于平台的一览](create-platform-specific-store-listings.md)，并仅包括该应用磁贴图标。

向 Windows 10 客户显示你的一览时，还可以选择防止应用商店使用应用软件包中的徽标图像，而是仅让应用商店使用你上传的图像。 这可以使你更好地控制应用在 Windows 10 的整个应用商店的各种显示中的外观。

若要在 Windows 10 的应用商店中仅使用上传的图像进行显示，请选中**对于 Windows 10 客户，显示上传的徽标图像，而不是包中的图像**框。

选中此框后，名为**上传的应用商店徽标**的新部分随即显示。 在这里可以上传 3 张图像，其中包括 300 x 300“应用磁贴图标”大小（如果选中此框，提供该图像的字段将移动到此部分中）。 如果你使用此选项，我们建议提供所有三种图像大小：300 x 300、150 x 150 和 71 x 71 像素。 但是，只需要 300 x 300 大小的图像。


## <a name="additional-art-assets"></a>其他插图资源

本部分允许你提供插图，从而帮助你在应用商店中更有效地展示你的产品：促销图像、Xbox 图像、可选的促销映像和预告片。 我们建议提供这些图像来创建更加吸引人的应用商店一览。 如果计划在应用商店一览中包括[视频预告片](#trailers)，则特别推荐 **16:9“hero”（1920 x 1080 或 3840 x 2160 像素）**；如果不包括，则一览顶部不会显示预告片。

若要添加这些图像，请在**其他插图资源**部分中选择**显示详细信息**。


## <a name="promotional-images"></a>促销图像

此部分中的图像可帮助改善一览的外观，并且有助于我们为你的应用考虑特色促销机会。 请注意，提供这些图像并不能保证你的应用将获得特别推荐。 有关详细信息，请参阅[轻松推广你的应用](make-your-app-easier-to-promote.md)。

下面是在设计促销插图时需要牢记的一些技巧：

- 图像必须为 .png 格式。
- 选择与应用、驱动器识别和差异化有关的动态图像。 避免使用图片库里的图像或通用的视觉效果。
- 请勿包含文本（你的品牌标记除外）。
- 尽可能减少图像中的空白空间。
- 避免显示你应用的 UI，并且请勿使用任何特定于设备的图像。
- 避免使用政治和国家/地区主题、标志或宗教符号。
- 请勿提供包含以下内容的图像：无礼手势、裸体、赌博、货币、毒品、烟草或酒精。
- 请勿使用指向观众的武器或过度暴力和血腥的内容。

**16:9“hero”（1920 x 1080 or 3840 x 2160 像素）**适用于所有 Windows 10 设备类型的应用商店中的各种布局。 无论你的应用针对哪种操作系统版本或设备类型，我们都建议提供此图像。 如果一览包括[视频预告片](#trailers)，则*需要*使用此图像进行正确显示。 对于 Windows 10 版本 1607 或更高版本的客户，它用作应用商店一览顶部的主图像（或在任何预告片播放完成后出现）。 

**2:1 (2400 x 1200)** 图像大小仅适用于应用支持全息设备系列的情况。

设计图像时，请牢记，在某些布局中，我们会在底部的三分之一处进行渐变处理，以便可在图像上清晰地显示市场营销文本。 因此，请确保避免在底部的三分之一处放置文本和关键视觉元素。 此外，我们还可能会裁剪你的图像，以便将你应用的品牌和最重要的细节内容放置在中心位置。

## <a name="xbox-images"></a>Xbox 图像

如果你向 Xbox 发布你的应用，则建议正确地显示这些图像。 

可以上传 3 种不同大小的图像：
- **品牌关键插图，584 x 800 像素**：在 Xbox.com 应用商店中使用。 必须包含产品的标题。 
- **主角人物，1920x1080 像素**：必须包含产品的标题。
- **特色促销，1080 x 1080 像素**：在 Xbox 应用商店中使用。 必须包含产品的标题。

> [!NOTE]
> 要在 Xbox 上获得最佳显示效果，还必须在[应用商店徽标](#store-logos)部分中提供 **9:16（720 x 1080 或 1440 x 2160 像素）**图像。


## <a name="optional-promotional-images"></a>可选的促销图像

如果应用支持以前的操作系统版本 (Windows 8.x 和/或 Windows Phone 8.x)，则必须提供这些图像，以便我们在促销布局中考虑特别推荐你的应用（但它们并不能保证你的应用将获得特别推荐）。 如果应用不支持这些以前的操作系统版本，可以跳过此部分。

**对于 Windows Phone 8.1 及更早版本**，促销布局中可使用两种图像大小：**1000 x 800 像素 (5:4)** 和 **358 x 358 像素 (1:1)**。 如果你的应用运行在 Windows Phone 8.1 或更早版本上，出于促销考虑，我们建议提供这两种大小的图像。  

> [!TIP]
> 对于任何支持 Windows Phone 8.1 或更早版本的提交，请务必在[应用商店徽标](#store-logos)部分中提供 300 x 300 的应用磁贴图标图像。 这将确保你的应用不会在应用商店中显示为空白图标。  

**对于 Windows 8.1 及更早版本**，某些促销布局可以使用大小为 **414 x 180** 像素的图像。 如果你的应用运行在 Windows 8.1 或更早版本上，出于促销考虑，我们建议提供此大小的图像。


## <a name="trailers"></a>预告片

预告片是一个简短视频，可为客户提供在实际操作中查看产品的方法，以便他们能更好地了解产品内容。 它们会显示在应用的应用商店一览顶部（前提是在[促销图像](#promotional-images)部分中包括 **1920 x 1080 像素的图像 (16:9)**）。 

预告片采用[平滑流式处理](http://www.iis.net/downloads/microsoft/smooth-streaming)进行编码，可根据客户端的可用带宽和 CPU 资源实时调整传送到客户端的视频流质量。

> [!NOTE]
> 预告片仅向 Windows 10 版本 1607 或更高版本的客户显示。

### <a name="upload-trailers"></a>上传预告片

你可以将最多 15 支预告片添加到应用商店一览中。 请确保每支预告片都满足下面列出的要求。

必须为每支预告片提供视频文件（.mp4 或 .mov）、缩略图图像和标题。

> [!IMPORTANT]
> 使用预告片时，必须在[促销图像](#promotional-images)部分中提供 **1920 x 1080 像素的图像 (16:9)**，以使预告片显示在应用商店一览顶部。 预告片播放完成后将显示此图像。

遵循以下建议，以使你的预告片有效：
- 预告片质量应较高，并为最小长度(建议为 2 分钟或更短)。 
- 帧率和分辨率应匹配源材料。 例如，以 720p60 拍摄的内容应按 720p60 进行编码和上传。 
- 为每支预告片使用不同的缩略图，以便让客户知道它们是唯一的。
- 由于某些布局会略微裁剪预告片的顶部和底部，因此请确保关键信息显示在屏幕的中央。

还必须遵循以下所述的要求。

**将预告片添加到你的应用商店一览中：**
1. 在指示框中上传预告片**视频文件**。 如果你想重新使用已上传的预告片（可能是不同语言的应用商店一览），也会显示一个下拉框。
2. 上传预告片后，你需要上传与之相应的**缩略图**。 略缩图必须为 1920 x 1080 像素的 .png 文件，通常应为取自预告片的静态图像。
3. 单击铅笔图标,为预告片添加**标题**（255 字符或更少）。
4. 如果想要为应用商店一览添加更多预告片，请单击**添加预告片**，重复上述步骤。

若要删除预告片，请单击其文件名旁边的 **X**。 你可以选择仅将其从当前应用商店一览中删除，或将其从产品（即仅针对此语言或所有语言）的所有应用商店一览中删除。

### <a name="trailer-requirements"></a>预告片要求

提供你预告片时，请确保遵循这些要求：

- 视频格式必须为 MOV 或 MP4。 
- 视频时长必须小于 30 分钟。 
- 预告片的文件大小不能超过 10 GB。 
- 缩略图必须为 PNG 文件，分辨率为 1920 x 1080 像素。 
- 标题不能超过 255 个字符。 

与应用商店一览页中的其他字段相似，预告片必须通过认证才能发布到应用商店中。 请确保预告片遵守[Windows 应用商店策略](https://msdn.microsoft.com/library/windows/apps/dn764944.aspx)。

还有其他要求，具体取决于文件类型。
