# <a name="windows-premium-ads-publishers-program"></a>Windows 高级广告发布者计划

作为连续努力帮助赚钱广告通过其应用程序的 Windows 开发人员的一部分，我们提供了[Windows Premium ads 的关系的发布者计划](http://www.windowspremiumapps.com)。 这是 curated 的一 ad 启用应用程序和使用情况、 项目和体验方面的类中最的游戏。 这些应用程序和游戏是可视性认证且具有识别正在高度的用户的世界各地的访问群体。

从我们的合作伙伴 ad 网络广告商可以单独标识这些应用程序和服务以独占方式到最高的投资这些应用程序的高级、 高收益广告于他们的营销活动的投资回报。 面向开发人员，这意味着属于此程序的应用程序可以获得更多广告收入。

## <a name="how-does-microsoft-add-apps-to-this-program"></a>如何 Microsoft 中添加应用程序对此程序？ 

我们不断查看针对[合格条件](#what-are-the-criteria-for-apps-in-the-program)存储区中的应用程序。 符合合格条件的应用程序都经过严格的手动审阅过程顺利而又明确定义用户体验检查并确保 ads 也放置以避免无意单击。 清除自动和手动审查过程的应用程序将添加到该程序。

这是当前仅邀请的程序，并只是会议合格条件不自动表示有享受成为 premium 计划的一部分的应用程序。 如果您想要了解有关提高您的应用程序体验，若要成为合格的程序的详细信息，请访问对我们在aiacare@microsoft.com。

## <a name="what-are-the-criteria-for-apps-in-the-program"></a>相关应用程序中的条件是什么？

在此程序的应用程序必须满足以下条件：

* 最小化保证每日 ad 请求卷。 
* 高可视性的印象的速率。 
* 良好质量单击跨所有 ad 格式分配给最小单击-通过-费率。 
* 按照交互式广告局 (IAB) 标准 ad 大小和位置方法。 不隐藏 ads 的关系。
* 全新的条幅广告的位置 （例如，广告不是彼此相邻的）。
* 跨多个应用程序不使用 ad 单位。
* 显示到其中的视频和/或间隙标题 ad 点[清除定义的用户流](https://blogs.windows.com/buildingapps/2017/08/31/best-practices-using-video-ads-windows-apps/)。 视频的广告不预缓存。 
* 控制 ad 刷新行为，以便从中介服务器返回的广告始终播放。
* 应用程序说明、 高保真屏幕截图和上载到存储的图标，请清除 great 存储状态。 强烈建议视频的尾部。
* 没有欺诈活动。

基于常用以确保它们继续满足合格条件监视程序中的应用程序。 要保持最小的每日 ad 请求量，请确保您的应用程序使用不会随着时间的推移会显著删除。 通过[运行常规升级市场活动](https://developer.microsoft.com/en-us/store/promote-your-apps)好方法执行此操作。
